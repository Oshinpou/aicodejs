Lesson 4.7: Best Practices in Using Conditional Statements

Why Best Practices Matter

Efficient and clean conditional logic makes your AI code more readable, maintainable, and less error-prone. Let's explore key practices for using if, else, switch and ternary operations effectively in JavaScript AI programs.

1. Keep Conditions Simple

if (isAIReady) {
  initializeAI();
}

Prefer short and descriptive conditions. Break complex logic into functions.

2. Use Truthy/Falsy Checks

if (userInput) {
  process(userInput);
}

JavaScript treats values like null, undefined, 0, "" as falsy. Use this for concise conditions.

3. Prefer `switch` for Multiple Values

switch (aiMode) {
  case 'train': trainAI(); break;
  case 'test': testAI(); break;
  case 'run': runAI(); break;
  default: idle();
}

Instead of many else if, switch keeps the logic clear for discrete values.

4. Use Ternary for Simple Assignments

let response = isAIOnline ? "Online" : "Offline";

Use ternary operators only when it improves clarity, not complexity.

5. Avoid Deep Nesting

// Instead of nested conditions:
if (user) {
  if (user.isActive) {
    if (user.isVerified) {
      grantAccess();
    }
  }
}

Refactor to:

if (user && user.isActive && user.isVerified) {
  grantAccess();
}

Summary

  • Write readable and clean conditionals
  • Refactor deep logic
  • Use ternary operators only when beneficial
  • Switch is best for checking multiple values
