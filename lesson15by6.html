Lesson 15.6 — Hoisting in Scope

Understand how JavaScript lifts declarations for smarter AI behavior.

What is Hoisting?

Hoisting is JavaScript’s default behavior of moving declarations to the top of the current scope (function or global).

Example: Hoisting with Variables

console.log(brain); // undefined
var brain = "AI Core";

JavaScript interprets it as:

var brain;
console.log(brain); // undefined
brain = "AI Core";

Hoisting with Functions

aiGreet(); // Works because function is hoisted

function aiGreet() {
  console.log("Hello from AI!");
}

Note: Function declarations are fully hoisted, while function expressions are not.

let and const Are Not Hoisted Like var

console.log(mode); // ReferenceError
let mode = "Learning";

Variables declared with let and const are in a “temporal dead zone” until their declaration is encountered.

Why Hoisting Matters in AI

In AI models where data flow and processing order matter, hoisting can impact when variables and functions are available. Mastering it helps avoid bugs and ensures reliable AI logic flow.
