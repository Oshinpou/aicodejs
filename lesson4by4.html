Lesson 4.4: Debugging

Learn how to debug your JavaScript code using console, breakpoints, and debugging tools.

What You'll Learn:

  • How to use console.log to trace issues in your code
  • Using breakpoints in browsers to inspect code execution
  • Debugging techniques for effective problem solving

Debugging is a crucial skill in development. In this lesson, we will explore techniques that help in identifying and fixing bugs in your JavaScript code.

Using console.log for Debugging

One of the simplest ways to debug JavaScript code is by using the console.log method to print variable values and track code execution:


let x = 10;
let y = 20;
console.log("The value of x is:", x); // Output: The value of x is: 10
console.log("The value of y is:", y); // Output: The value of y is: 20

Using console.log at various points in your code helps you understand how values change and can pinpoint where the issue arises.

Setting Breakpoints in Developer Tools

In most browsers, you can set breakpoints in the Developer Tools to pause code execution and inspect variable states:

  1. Right-click on your code in the browser's developer tools and select "Add Breakpoint".
  2. Once the breakpoint is hit, the code will pause, allowing you to inspect variable values and step through the code.
  3. This lets you see exactly where things are going wrong.

Common Debugging Techniques

  • Print variable values with console.log().
  • Use breakpoints to pause and step through code.
  • Check for syntax errors or typos that could be causing issues.
  • Use debugger; statements in your code to invoke breakpoints programmatically.

These techniques are incredibly useful for understanding how your code executes and resolving any errors you might encounter.
