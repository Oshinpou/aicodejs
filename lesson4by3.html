Lesson 4.3: Error Handling

Learn how to gracefully handle errors in JavaScript using try, catch, throw, and finally blocks.

What You'll Learn:

  • How to handle errors with try and catch
  • Throwing custom errors using the throw statement
  • Executing code after try/catch using finally

Error handling is essential to building robust applications, especially when working with AI and real-time data processing.

Try and Catch

The try-catch statement allows you to test a block of code for errors and handle those errors if they occur:


try {
  let result = riskyFunction();
} catch (error) {
  console.log("Error occurred: " + error.message);
}

If an error occurs inside the try block, the catch block will run and log the error message.

Throwing Errors

You can throw custom errors using the throw statement. This is useful when you want to manually handle an error condition:


function checkAge(age) {
  if (age < 18) {
    throw new Error("Age must be 18 or older");
  }
  return "Age is valid";
}

try {
  console.log(checkAge(16)); // Will throw an error
} catch (error) {
  console.log(error.message); // Output: "Age must be 18 or older"
}

In this example, an error is thrown when the age is less than 18, and the error message is caught and displayed in the catch block.

Finally Block

The finally block runs regardless of whether an error occurs or not. It's often used for cleanup actions:


try {
  let result = riskyFunction();
} catch (error) {
  console.log("Error occurred: " + error.message);
} finally {
  console.log("This will always run, regardless of errors.");
}

In this example, the finally block will execute after the try-catch statement, regardless of an error.
